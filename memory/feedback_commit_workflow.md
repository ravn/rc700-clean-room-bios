name: Commit workflow - investigate problems first
description: Before committing, investigate any problems found during the session, create tasks/issues, then summarize and commit

---

When preparing to commit, follow this workflow:
1. Investigate any problems discovered during the session
2. Create tasks and issues as necessary for unresolved items
3. Summarize work and findings in the project
4. Then commit

**Why:** User wants problems addressed proactively, not deferred silently. The commit should reflect a clean state with known issues tracked.

**How to apply:** Before every commit, review the session for warnings, test failures, TODOs, or design concerns. Fix what can be fixed, file the rest as tasks/issues, then commit with a summary.
